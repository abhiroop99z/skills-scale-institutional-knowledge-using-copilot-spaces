# OctoAcme — Cross-Functional Handoff Checklist

Use this checklist at phase boundaries to ensure requirements, ownership, and quality expectations are explicit before work moves forward.

## Requirements Ready (Initiation -> Planning)
- [ ] Problem statement, scope boundaries, and success metrics are documented
- [ ] Business Analyst has captured core requirements and assumptions
- [ ] Product Manager has defined initial acceptance criteria
- [ ] Stakeholder Reviewer(s) for the initiative are identified
- [ ] Open questions, risks, and dependencies are logged with owners

## Design Ready (Planning -> Build)
- [ ] UX Designer has provided flows/wireframes/prototypes for in-scope work
- [ ] Accessibility and usability considerations are documented
- [ ] Product + UX review completed and design decisions recorded
- [ ] Engineering and QA questions on design behavior are resolved

## Dev Ready (Planning -> Execution)
- [ ] Backlog items are prioritized and estimated
- [ ] Acceptance criteria are testable and unambiguous
- [ ] Technical dependencies and environment constraints are identified
- [ ] Delivery ownership is clear across PM, Scrum Master, and Engineers
- [ ] Definition of Done and non-functional expectations are agreed

## QA Ready (Execution -> Validation)
- [ ] QA has test scenarios mapped to acceptance criteria
- [ ] Business Analyst traceability from requirements to test cases is present
- [ ] UX validation checks (behavior/accessibility) are included
- [ ] Defect triage owners and severity rules are agreed
- [ ] Exit criteria for release candidate are explicit

## Release Ready (Validation -> Release)
- [ ] CI/security checks, smoke tests, and rollback plan are complete
- [ ] Stakeholder review/UAT outcomes are documented
- [ ] Product Manager confirms scope/value readiness
- [ ] Project Manager confirms release communication and timing
- [ ] Final go/no-go decision and approvers are recorded
